# Shared Offline Basemap

NukeMap and VIPTrack now share one richer offline basemap instead of each using a separate lightweight cache.

## What Changed

- Both workspaces use the same local storage cache key: `nomad-offline-atlas-cache`
- Both workspaces install or refresh the basemap when they launch/onboard if the cache is missing or outdated
- The bundled source atlas lives at [web/nukemap/data/offline_atlas.json](C:/Users/--/repos/project-nomad-desktop/web/nukemap/data/offline_atlas.json)
- The atlas is generated by [tools/build_offline_atlas.py](C:/Users/--/repos/project-nomad-desktop/tools/build_offline_atlas.py)

## Basemap Contents

The generated atlas currently includes Natural Earth 1:50m data for:

- land polygons
- coastlines
- country borders
- admin-1 borders (state/province outlines)
- lakes
- rivers
- populated places

## Runtime Flow

1. NukeMap or VIPTrack launches.
2. The workspace checks `nomad-offline-atlas-cache`.
3. If the cache is missing or the atlas version changed, it fetches `/nukemap/data/offline_atlas.json`.
4. The downloaded atlas is cached once and reused by both workspaces.
5. Light and dark themes restyle the same atlas so both tools stay visually aligned with NOMAD themes.

## Reset / Reinstall

If you need to force a clean reinstall of the shared basemap:

- remove `localStorage['nomad-offline-atlas-cache']`
- optionally clear onboarding flags such as `nukemap-welcomed` and `viptrack_onboarded`
- relaunch NukeMap or VIPTrack so the basemap is downloaded again
